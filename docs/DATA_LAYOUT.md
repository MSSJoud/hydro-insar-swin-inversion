# Data layout

The software repo is designed for mounted data directories rather than shipping large datasets inside the image.

Recommended layout:

```text
repo-root/
  data/
    raw/
    derived/
  outputs/
```

Inside the container this becomes:

```text
/workspace     -> repository
/data/raw      -> raw input datasets
/data/derived  -> derived or aligned products
/workspace/outputs -> outputs generated by the run
```

## Intended categories

### `/data/raw`

Examples:

- raw MintPy products
- raw W3RA products
- downloaded GRACE, SMAP, SWOT files
- groundwater well source files

### `/data/derived`

Examples:

- overlap products
- aligned multisensor bundles
- processed well-validation tables

### `/workspace/outputs`

Examples:

- figures
- summary tables
- exported notebooks
- lightweight result summaries

## Current note

Some legacy research scripts still expect hard-coded paths from the original working environment. Those are being migrated toward config-driven path handling in this software repository.
